# CCTV直播 — 架构与实现说明

把央视官网 `https://tv.cctv.com/live/` 与央视频 `https://www.yangshipin.cn` 的直播网页**套壳**进一个
Android App，做成可侧载到 Android TV 盒子 / 平板的「打开即自动选台 + 自动全屏播放」的遥控器友好应用。

自动化思路移植自 [`../CCTV_Viewer`](https://github.com/eanyatonic/CCTV_Viewer)（致谢见 README）。本文档详尽到
**可据此从零重建本 App**。

---

## 1. 核心设计取舍

1. **选台不靠模拟点击**：每个频道就是一个独立 URL，直接 `loadUrl/loadUri` 即可。频道清单见
   `data/Channels.kt`（41 个）。
2. **唯一需要"点"的是播放器的全屏按钮**：注入 JS 轮询等按钮出现再 `.click()`，并强制开声音、保活播放。
3. **浏览器引擎做成可替换层**（`web/WebPlayer.kt` 的 `WebEngine` 枚举 + `ACTIVE_ENGINE`）：
   - **系统 WebView**（`web/SystemWebPlayer.kt`）：自带 `evaluateJavascript`，体积最小，最稳。
   - **GeckoView**（`web/GeckoWebPlayer.kt`）：Firefox 引擎打包进 APK，跨设备一致、自带 Widevine DRM。
   - 切换只改 `WebPlayer.kt` 里的 `ACTIVE_ENGINE` 一行。**当前默认 = GeckoView。**

## 2. 构建与 SDK

- AGP **8.7.3** / Kotlin **2.0.21** / Gradle **8.11.1**；`compileSdk 35`、`minSdk 24`、`targetSdk 34`，JVM 1.8。
- Jetpack Compose（BOM 2024.09）+ Material3 + Navigation-Compose。无 ExoPlayer / 网络栈（套壳不需要）。
- **GeckoView**：`org.mozilla.geckoview:geckoview:136.0.20250317200840`。
  - 发布在 **Mozilla 自家 Maven**（`settings.gradle.kts` 里加 `maven { url = uri("https://maven.mozilla.org/maven2") }`），**不在 Maven Central**。
  - **版本被工具链锁死在 ≤136**：≥140 拉 `kotlin-stdlib` 2.1+（当前 Kotlin 2.0 编译器读不了其元数据，内部错误）；
    ≥147 拉 `androidx.core` 1.17 / `media3` 要 `compileSdk 36` + AGP 8.9。要上新版须先整体升级工具链。
  - 配套：`androidx.core:core-ktx:1.15.0`（对齐 GeckoView 拉入的 core 1.15.0）。
- **按 ABI 分包**（`app/build.gradle.kts` 的 `splits.abi`，`isUniversalApk = false`）：
  GeckoView 每架构约 +40MB，故 `armeabi-v7a`（32 位盒子，~154MB）、`arm64-v8a`（64 位平板/盒子，~187MB）各出一个。
- 构建：`./gradlew :app:assembleDebug`，产物在 `app/build/outputs/apk/debug/app-<abi>-debug.apk`。
- 侧载：`adb install -r app-arm64-v8a-debug.apk`（盒子先开开发者模式 + 允许未知来源）。
- 本机 SDK 路径写在 `local.properties` 的 `sdk.dir`（不提交，已 gitignore）。

> 若想要轻量的 WebView-only 版本（~18MB 单包）：移除 geckoview 依赖与 `GeckoWebPlayer.kt`、
> `assets/extensions/cctv`、Mozilla Maven 仓库、`splits` 块，把 `compileSdk` 回 34、`core-ktx` 回 1.12.0，
> 并把 `ACTIVE_ENGINE` 设为 `SYSTEM_WEBVIEW`。

## 4. 频道数据（`data/Channels.kt`）

```kotlin
enum class ChannelGroup(val title: String) { CCTV("央视频道"), LOCAL("地方频道") }
enum class PlayerKind { CCTV, YANGSHIPIN }   // 决定全屏按钮选择器（见 §7）
data class Channel(val id: Int, val name: String, val url: String, val kind: PlayerKind, val group: ChannelGroup)
```

- **央视 20 个**（`kind = CCTV`，`group = CCTV`）：URL 形如 `https://tv.cctv.com/live/cctv1/`、`cctv2/`…；
  少数特殊：`cctvjilu`（纪录）、`cctvchild`（少儿）、`cctv5plus/`、`cctveurope`、`cctvamerica/`。
- **地方卫视 21 个**（`kind = YANGSHIPIN`，`group = LOCAL`）：URL = `https://www.yangshipin.cn/tv/home?pid=<PID>`，
  PID 是央视频的数字台号（如 北京 `600002309`、江苏 `600002521`…，逐字移植自 CCTV_Viewer）。
- `id` 即下标（0..40）；`prev(id)/next(id)` 到头循环；`groups` 给首页分区；`all` 全量。
- ⚠️ **地方台必须用桌面 UA**：`yangshipin.cn` 桌面页在移动 UA 下会跳移动版/引导下载、播放器不渲染 → 黑屏。
  两个引擎都强制桌面 Chrome UA（见 §5/§6）。

## 5. 系统 WebView 实现（`web/SystemWebPlayer.kt`）

Compose `AndroidView` 里放一个 `FrameLayout` 容器 + `WebView`。关键设置：

```kotlin
WebView.setWebContentsDebuggingEnabled(true)   // 桌面 Chrome chrome://inspect 可调试
settings.apply {
    javaScriptEnabled = true
    domStorageEnabled = true
    mediaPlaybackRequiresUserGesture = false        // 允许自动播放
    databaseEnabled = true
    cacheMode = LOAD_DEFAULT
    javaScriptCanOpenWindowsAutomatically = true
    useWideViewPort = false                         // 关宽视口/概览缩放，否则全屏后画面比屏幕大、可拖
    loadWithOverviewMode = false
    setSupportZoom(false); builtInZoomControls = false
    userAgentString = DESKTOP_USER_AGENT            // 见下
}
// 全屏适配：网页全屏按 1920×1080 设计，整体缩放到本机屏幕。播放恒横屏，按长边=宽、短边=高算。
val scale = round(min(longEdge/1920.0, shortEdge/1080.0) * 100).toInt()
setInitialScale(scale.coerceAtLeast(1))
```

- `DESKTOP_USER_AGENT`（`internal const`，两引擎共用）：
  `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/119.0.0.0 Safari/537.36`
- 注入：`onPageStarted` 注入 `CctvAutomation.fastLoadingJs`，`onPageFinished` 注入 `fullscreenJsFor(kind)`
  （两者当前同一段 `pageScript`，幂等）。用 `evaluateJavascript`。
- `WebChromeClient.onShowCustomView/onHideCustomView`：处理网页发起的 HTML5 真全屏，把 customView 叠满容器。
- 切台：`update` 块里目标 URL 与当前不同才 `loadUrl`，避免重复加载。`onDispose` 里 stop/loadUrl(about:blank)/destroy。

## 6. GeckoView 实现（`web/GeckoWebPlayer.kt` + `MyTvApp.kt`）

### 6.1 GeckoRuntime —— 必须懒加载、进程单例（关键坑）

`MyTvApp` 里 **不要** 在 `onCreate` 直接 `GeckoRuntime.create()`。改成懒加载、进程唯一：

```kotlin
class MyTvApp : Application() {
  companion object {
    @Volatile private var runtimeInstance: GeckoRuntime? = null
    fun geckoRuntime(ctx: Context): GeckoRuntime =
      runtimeInstance ?: synchronized(this) {
        runtimeInstance ?: createRuntime(ctx.applicationContext).also { runtimeInstance = it }
      }
    private fun createRuntime(app: Context): GeckoRuntime {
      val s = GeckoRuntimeSettings.Builder()
        .remoteDebuggingEnabled(true)   // 桌面 Firefox about:debugging 可连
        .consoleOutput(true).aboutConfigEnabled(true).build()
      val rt = GeckoRuntime.create(app, s)
      rt.webExtensionController.installBuiltIn("resource://android/assets/extensions/cctv/")
        .accept({ /*log*/ }, { /*log*/ })
      return rt
    }
  }
}
```

> **为什么必须懒加载**：若启动即 `create()`，会在**首页就拉起 Gecko 的前台服务子进程，而屏幕上没有可见的
> GeckoView**——三星 One UI 等会把这种"无可见 UI 的前台服务/多进程"判为滥用，于是一进播放页约 3 秒就把整个
> 前台 App `SIGKILL`（前台 TOP 进程、无 native 崩溃/minidump/Java 异常）。改成进 `GeckoWebPlayer` 时才
> 创建后，Gecko 进程随可见播放页一起起来，不再被杀。

### 6.2 Session 配置与回调

```kotlin
val settings = GeckoSessionSettings.Builder()
    .usePrivateMode(false)
    .userAgentOverride(DESKTOP_USER_AGENT)              // 只覆盖 UA 字符串（地方台需要桌面页）
    .viewportMode(GeckoSessionSettings.VIEWPORT_MODE_MOBILE)  // 移动视口
    .build()                                           // ⚠️ 不要设 userAgentMode=DESKTOP，否则按桌面渲染、关掉移动视口缩放
val session = GeckoSession(settings)
session.navigationDelegate  = …onCanGoBack → 回调上层（返回键用）
session.contentDelegate     = …onCrash/onKill → 内容进程崩溃/被杀时重开 session 并重载当前 URL
session.permissionDelegate  = …onContentPermissionRequest → 对 AUTOPLAY_AUDIBLE/INAUDIBLE 与
                                MEDIA_KEY_SYSTEM_ACCESS(Widevine) 返回 VALUE_ALLOW，其余 null
session.open(MyTvApp.geckoRuntime(context))
view.setSession(session); session.loadUri(channel.url)
```

- 注入：GeckoView **没有 `evaluateJavascript`**，靠 §6.1 安装的内置 WebExtension 内容脚本（`assets/extensions/cctv/`）。
- 切台：`update` 块里 URL 变了才 `loadUri`。`onDispose` 里 `setActive(false)` + `close()`。

## 7. 注入脚本逻辑

两套等价逻辑：WebView 用 `CctvAutomation.pageScript`（Kotlin 字符串，`evaluateJavascript` 注入）；
GeckoView 用 `assets/extensions/cctv/cctv.js`（WebExtension 内容脚本）。共同行为：

1. `window.__cctvAuto` 守卫，保证同一文档只跑一个轮询。
2. **精简页面**（加速盒子首屏/切台）：清空所有 `<img>.src`；清掉含 `login/index/daohang/grey/jquery` 的脚本；
   清空 `newmap/newtopbz/newtopbzTV/column_wrapper` 等容器 innerHTML。
3. **轮询点全屏**：每 16ms 找 `#player_pagefullscreen_yes_player`（央视官网）或 `.videoFull`（央视频），
   出现即 `.click()` 进入网页全屏，**只点一次**（进入后该"进入"按钮消失，自然不会再点）。
4. **保活媒体**：`video.muted=false; video.volume=1; 若 paused 则 play()`；点中后降到每 500ms 一次，约 10 分钟后停。

### 全屏适配（两引擎做法不同，都与分辨率无关）

- **WebView**：用 `setInitialScale(min(屏宽/1920, 屏高/1080)×100)` 把整页缩放到本机屏幕（§5）。
- **GeckoView**：Gecko 里**改 viewport meta 不生效**（实测 `innerWidth` 不随注入变化），改用 `cctv.js` 注入 CSS
  把 `<video>` 钉成铺满视口、固定不动、禁页面滚动：

  ```css
  html,body { overflow:hidden!important; margin:0!important; width:100%!important; height:100%!important }
  video {
    position:fixed!important; top:0!important; left:0!important; right:0!important; bottom:0!important;
    width:100vw!important; height:100vh!important; max-width:none!important; max-height:none!important;
    object-fit:contain!important; z-index:2147483647!important; background:#000!important; margin:0!important;
  }
  ```

  用相对单位 `100vw/100vh` + `object-fit:contain`，**任何分辨率**（平板、盒子接不同电视）都自适配铺满、保持比例。
  （电视 overscan 过扫描属电视端"画面尺寸"设置，与本代码无关。）

## 8. 遥控器 / 交互逻辑

### 按键分发（`MainActivity`）

```kotlin
class MainActivity : ComponentActivity() {
  var keyHandler: ((KeyEvent) -> Boolean)? = null      // 播放页注册
  override fun dispatchKeyEvent(e: KeyEvent): Boolean =
      if (keyHandler?.invoke(e) == true) true else super.dispatchKeyEvent(e)
  // onCreate: enableEdgeToEdge() + setContent { NavHost(home → player) }
}
```

`dispatchKeyEvent` 在 Activity 层**优先**截获按键，先于 WebView/GeckoView 默认处理。

| 场景 | 按键 | 行为 |
|---|---|---|
| 首页 | 方向键移动 + OK | 在频道网格间移动焦点、进入播放（`tvClickable` 焦点高亮，初始焦点在 id=0） |
| 播放页 | 上 / CHANNEL_UP | 上一台（`Channels.prev`，同一 player 内重载） |
| 播放页 | 下 / CHANNEL_DOWN | 下一台（`Channels.next`） |
| 播放页 | 返回 | 网页可后退则先后退，否则回首页 |

> 平板无方向键：用触摸点频道进播放页，退出后重新选台即可（不做触摸切台）。盒子用遥控器方向键。

### 播放页（`ui/player/PlayerScreen.kt`）

- 进场：`requestedOrientation = SCREEN_ORIENTATION_SENSOR_LANDSCAPE`、`FLAG_KEEP_SCREEN_ON`、沉浸式隐藏状态/导航栏；
  离场 `onDispose` 全部还原。
- `activity.keyHandler` 注册「上/下切台」；`BackHandler` 实现「先网页后退、否则回首页」。
- 切台/进场浮出频道名 2 秒（`AnimatedVisibility`）。
- `WebPlayer(channel, engine = ACTIVE_ENGINE, …)`：铺满黑底 `Box`。

### 首页（`ui/home/HomeScreen.kt`）

- `LazyVerticalGrid`（`GridCells.Adaptive(168.dp)`），顶部标题 + 央视/地方两个分区标题（跨整行 span）。
- 每个频道一张 `ChannelCard`（红色 LIVE 徽标 + 台名），`tvClickable` 焦点高亮，OK → `onChannelClick` 导航到 `player/{id}`。

## 10. AndroidManifest 关键项

```xml
<uses-feature android:name="android.hardware.touchscreen" android:required="false"/>
<uses-feature android:name="android.software.leanback" android:required="false"/>  <!-- 否则 TV 桌面/商店不收 -->
<application android:name=".MyTvApp" android:banner="@drawable/banner"
             android:usesCleartextTraffic="true" …>
  <activity android:name=".MainActivity"
      android:configChanges="orientation|screenSize|smallestScreenSize|screenLayout|keyboard|keyboardHidden"
      android:exported="true">
    <intent-filter>
      <action android:name="android.intent.action.MAIN"/>
      <category android:name="android.intent.category.LAUNCHER"/>
      <category android:name="android.intent.category.LEANBACK_LAUNCHER"/>  <!-- TV 桌面图标 -->
    </intent-filter>
  </activity>
</application>
```

> ⚠️ `configChanges` 必须含 `orientation|screenSize`：播放页强制横屏会触发朝向变化，没有它 Activity 会被销毁重建，
> GeckoView 对 Activity 重建很脆弱。

## 11. 已知边界

- **DRM / 地理封锁**：CCTV5 等体育频道版权/加密最严，海外有地理封锁，需大陆网络；GeckoView 自带 Widevine，
  首播会联网下载 CDM。这是 GeckoView 相比系统 WebView 的主要价值。
- **设备差异**：系统 WebView 在老盒子上内核版本不一；若表现不一致或 DRM 失败，用 GeckoView（当前默认）。
- **调试**：WebView 用桌面 Chrome `chrome://inspect`；GeckoView 用桌面 Firefox `about:debugging`
  （已开 `remoteDebuggingEnabled`）。
- **法律边界**：自家盒子上看没问题；不要分发 apk、不要商用。
